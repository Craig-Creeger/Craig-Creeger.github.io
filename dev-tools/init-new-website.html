The way you choose to start a new site is highly opinionated and variable depending on your needs. This is one way to get started with a website that uses a SASS pre-processor and ES6 style Javascript.

  1. Install your build tools: Node, npm, Bower, Sass, and Grunt
  2. Create the root directory of your new site. Ex: mkdir ~/Sites/newSite
  3. Create Bower’s bower.json file with: bower init using all of the defaults except private package.
  4. Install these components.
    1. bower install --save normalize.css
    2. bower install --save html5shiv
    3. bower install --save Respond
    4. bower install https://github.com/h5bp/html5-boilerplate.git
      Note: We are not saving h5bp because we are going to pull the main.css file out and modify it.
  5. Create npm’s package.json file with npm init
  6. Install your npm packages:
    1. See the Grunt page for some packages to add.
  7. Configure Grunt.
    1. Create a Gruntfile.js file and edit it.

      hello

npm Build Option

I think this might now be my prefered method.

  1. Create a directory and vhost
  2. Do an npm init
  3. Install these packages to your site (npm install --save-dev packageName):
    1. babel
    2. node-sass
    3. jquery (use --save) if you need jQuery
  4. Install these packages globally (npm install -g packageName):
    You can use npm -g list --depth=0 to find out if they are already installed.
    1. browser-sync
    2. autoprefixer
    3. postcss-cli
  5. Create your directory structure:
    1. components
      1. es6
      2. sass
        1. partials
      3. tmpcss
    2. dist
      1. css
      2. js
    3. node_modules
  6. Add a .gitignore file 
    node_modules/
bower_components/

Automatically Adding CSS Vendor Prefixes

I tried Autoprefixer, but ran into some problems. Below are the pieces to Autoprefixer.

postcss

This tool will transform CSS and it is very fast. It does the same job as SASS, but it uses its own syntax I think. You will not need this, but it may be worth exploring whether or not this is a better way to go (over SASS).

postcss-cli

I would like to have a method to “watch” for file changes. You will need the postcss-cli to do that. It will also accept:

  • Output directory
  • Input files (you can use wildcards such as /css/*.css)
  • The type of transformation you want (i.e. Autoprefixer)
  • Location to a config file. This config file can specify the Autoprefixer options; see below.
  • Whether you want to Watch or not.

autoprefixer

This is the routine that performs CSS vendor prefixes transformations. It will accept the options:

  • browsers: 'last 2 version' (for example
  • cascade: true (default to make pretty)
  • add: true (defaults to adding prefixes)
  • remove: true (defaults to removing prefixes)

If I didn’t need the watch functionality provided by postcss-cli, then I could simply use just autoprefixer.

autoprefixer-core

The main logic for Autoprefixer is in this package. It contains a function called process(css, options) which is called, I think, from autoprefixer.

Finally, this is where I got stuck.

The options going into the process() function are:

  • from: file path of source CSS
  • to: file path of destination
  • safe: enables safe mode (default is false)
  • map {
        inline:false,
        prev: original source map
    }

I would really like to get the CSS maps updated, but I can’t figure out how to pass the map option in. The maps options are little more complicated as described in the documentation.

Update: I got unstuck.

The Autoprefixer will handle maps by default IF they are inlined (embedded in the CSS via a comment), so I changed by SASS build step to inline the maps instead of creating a separate map file. Now all is good.
