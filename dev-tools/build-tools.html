Build Tools

npm Scripts

Since you are already using npm, this is a great option because no additional tools to install and the configuration is easy. A couple of articles to get you started:

  • http://blog.keithcirkel.co.uk/how-to-use-npm-as-a-build-tool/
  • http://www.sitepoint.com/guide-to-npm-as-a-build-tool/

I have been using this method and really like it so far.

For a more advanced example that uses Javascript programs (via Node) to build your site, check out “You Might Not Need Gulp.js”.

Grunt

Now irrelevant. It’s slow and the configuration is ugly. Skip this one.

The First Big One

Many, many people are using Grunt. The install is easy and there are a bunch of plugins for it.

Cons

Writing the configuration section of Gruntfile.js is a pain and it gets slow if you are working on a large project.

My Experiments

I have just gotten started with Grunt.

Broccoli

This is used by EmberJS. Suppose to be fast, but the document is terrible and not many examples outside of Ember.

Gulp

This may now be more powerful than Grunt, but Webpack could make it obsolete.

Webpack

Depending on your preferences, Webpack could be the new shiny build tool.

Brunch

Want something newer and shinier? Brunch could be the answer. Give this one a try—the documentation is very good.
