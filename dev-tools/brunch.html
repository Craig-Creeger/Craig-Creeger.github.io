Spinning Up a New Website

There are only three Brunch CLI commands:

brunch new, which initializes a new project

brunch watch that recompiles/moves files that change.

brunch build that creates a final website for you.

Pre-requisites

  • Install Node 4.0 or higher
  • Install Bower globally.

Initialization

  1. Create folder where you want your project to be, say, ~/Sites/brunchTesting
  2. From the command line, navigate to that folder and then do:
    1. brunch new
      which will create a Node package (package.json), a Brunch directory structure, and base it on the default skelton (https://github.com/brunch/dead-simple).
  3. To get auto-reload-brunch to work I had to manually include its auto-reload.js file into my HTML as a script.
