Bower

Introduction

This might be on the decline now. I prefer to keep things as simple as possible, and since npm is pretty much a requirement I’ll just stick with that one.

Strictly a front-end only package manager. Extremely flexible in that it allows you to install Github repositories as package sources if they are not listed on the official Bower repository.

It handles modules regardless of what they contain. Whether that be; CSS, HTML, images or Javascript files. You also get the benefit of separating dependencies on the back-end from the front-end as well.

If the purpose of the framework or library you are wanting to include is mostly for front-end purposes like jQuery or Bootstrap, then sticking with Bower or equivalent is a good idea.

The upside of picking Bower, is that it works really well with task runners like Gulp and Grunt. You are able to achieve the same things as Browserify can (only for client-side libraries) through a few simple tasks you can find in various repositories and gists.

Official documentation

Spinning Up a New Site

Install Bower globally if that has not already been done.

  1. Create the root directory for your site.
  2. Create a new file called .bowerrc in the root. The contents of that file should specify the location where you want Bower to install packages. For example: 
    {
  "directory": "src/bower_components"
}

    If you don’t do this then the packages will be placed in a directory called bower_components in the root.

  3. Create a bower.json file by entering the command:
    bower init
  4. Start installing bower packages using the --save switch. For example:
    bower install --save colorbox

Common Commands

Initializing Bower

Before you add packages to your site, you will need to create a bower.json file first. Use the init command to do this:

bower init

Installing Packages

bower install endPoint

bower install --save endPoint //writes package to bower.json file (both Dev and Prod)

When saving to the bower.json file, bower will include the version number. You may want to double check that it specified it the way you want. Refer to the SemVer syntax if needed.

Updating Packages

bower update package --force-latest

Uninstalling Packages

bower uninstall package

Others

bower list //show installed packages and version
