Testing and Debugging Web Pages

Debugging CSS and Javascript

All browsers come with built-in tools (or add-on in the case of Firefox) for inspecting the DOM (generally to track down CSS problems) and stepping through your Javascript code.

Secrets of the Browser Developer Tools has a great list of how to find these tools and what can be done with them.

Some little nuggets I especially enjoyed are:

  • Use the Javascript statement debugger; to set a breakpoint in your code.
  • You can use Source Maps to debug minified code or CSS.
  • With Chrome, you can emulate a touch device.
  • Shift, Command (Cntl on Windows), Delete will delete your browser cache.
  • You can debug an iOS device by plugging it into your Mac and then bringing up developer tools in the Macâ€™s Safari.

The Webkit Inspector goes into more detail on debugging with Safari or Chrome.

Testing Your Website

Essential Tools for Testing Websites is a list of tools for testing performance, accessibility, and SEO.

  • Adobe BrowserLabs
  • Validing your CSS: CSS Validator or CSS Lint or Unicorn
  • Validing your HTML: HTML Validator by the W3C or Unicorn
  • Validating your Javascript: JSLint and JSHint for checking syntax and formatting
