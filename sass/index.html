SASS and Compass

The goal of SASS is to make reading and writing CSS easier. It also gives new functionality that CSS doesn’t have like performing math or using variables. This comes at the expense of having to learn Sassy CSS (SCSS) syntax and setting up some way to automatically compile the source files. Furthermore, debugging CSS won’t be quite as straight forward (although it does provide maps).

I don’t find CSS particularly difficult to read or write, so the only thing I would really miss is variables.

My recommendation: Ignore this technology unless the “pros” outweigh the “cons” for your situation.

Seven Excellent Websites for Learning SASS

Official Sites

SASS — http://sass-lang.com

SASS gives standard CSS some intelligence. For example variables, which are awesome for defining standard colors and schemes, functions (they call it mixins), compiled includes (vs a separate HTTP request), a nicer way to express dependent selectors (called nesting), inheritance (via the @extent), and computing values using mathematical expressions.

Compass — http://compass-style.org

Compass gives you a bunch of pre-build mixins and other functions. For example,

  • Text truncation with ellipses
  • Grid layout
  • Footer stick
  • Stretching an AP div to fill its container
  • Automatically calculate image width and height
  • Various color and math functions
  • Setting default border-radius and shadows
  • Adding vendor prefixs to box-sizing

Installation

Installation is super easy on a Mac. Just install the Compass Gem (gem install compass) and optionally the CSS Parser (gem install css_parser) so that you can get some statistics on your CSS file by issuing compass stats.

Now, create a new project using http://compass-style.org/install/, but this isn’t strictly necessary.

You can start a project with a "reset" stylesheet, but the one that comes with SASS kinda sucks (it screws with expected margins on paragraphs and bullets on lists). It would be better to @include the normalize.css instead.

Use with Dreamweaver

TLDR; It basically sucks with CS5.

  • Color coding of the .scss files is not build in.
  • Constantly get the message “This file has been modified outside of Dreamweaver. Do you want to reload it?” when your page is still open in the editor, which is usually.
  • You lose code hinting

Sure, there are some work arounds to make it less crappy – do some googling.

Workflow

If you would rather not compile your scss file each time it changes, you can have Compass watch your directory and automatically compile it each time it changes. The “watching” process will just keep running in your terminal window until you hit Cntl+c.

Partials

The partial files must start with an underscore, such as _colors.scss

When importing the partial, you do not need to type the underscore or the extension. i.e.

@import 'colors'
