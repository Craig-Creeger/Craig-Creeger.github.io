Box Model

Sitepoint has a good description of the Box Model.

Width and Height

You must understand how the overall width and height of a box is calculated if you need to specify exact dimensions of a box. By default, the width and height is applied to the “content” area of the box not including the padding, border, or margin. This behavior can be overridden with the CSS property box-sizing, which takes the values content-box | padding-box | border-box | inherit. Note: As of Fall 2013, Firefox still needs to be prefixed. -moz-box-sizing

The formula to remember is:

border + padding + content = total

Let’s say we are solving for the width property. In the above formula, border includes border-left and border-right.

Likewise, padding represents padding-left and padding-right. Content represents width. Total is the actual amount of space taken on the screen.

Example with Border

In this image, the width and height of Kermit is 345 x 517. Note: the thin white line is part of the jpeg image. He was placed on the page with:

<figure class="kermit"><p>Kermit the Frog</p></figure>

The image’s <div> was styled as follows:

.kermit {
   background-color:#ADCC54; /* green */
   background-image:url(kermit.jpg);
   background-repeat:no-repeat;
   background-position: left top; /* from padding */
   padding:20px; 
   border-style:dotted;
   border-width:30px;
   border-color:#BF3926; /* orange */
   width:305px; /* CONTENT width */
   height:477px; /* CONTENT height */
   }

An interesting thing to point out is the background-color is applied underneath the border. Because our border has “holes” in it, we can see the green colored background.

border + padding + content = total

This one is a bit tricky. We are solving for content width. The first thing is know is a background-image is placed in relation to the padding area (versus underneath the border area or content area). So, the total width of our figure is the left border (30px) + image width (345px) + right border (30px) = 405px.

Now that we have the total width, the formula is:

border + padding + content = total
(30*2) + (20*2) + content = 405px
content = 405 - 60 - 40
content = 305px width

Solving for the height is the same process.

Example without a Border

Probably a more common scenario is a box with no border and you need to space the text away from the edge.

Now the CSS is:

.kermit {
   background-image:url(kermit.jpg);
   background-repeat:no-repeat;
   
   padding:20px;
   
   width:305px; /* CONTENT width */
   height:477px; /* CONTENT height */
   }

border + padding + content = total

(20*2) + 0 + content = 345
40 + content = 345
345 - 40 = content
305px = content width

It’s facinating to note that the width and height values are exactly the same as in the previous example.

IE7 and older

IE browsers are notorious for not following web standards and one of the biggest violations, from a web developer’s perspective is IE7’s box model. It considers width and height to include the padding and border and it can not be changed with the box-sizing property because it doesn’t support it. There are several work-arounds for this problem, but hopefully you will never have to code to IE7. In the past, I’ve used the <div> inside another <div> hack.

Div inside Div Hack

The outer <div>’s width and height should match the size of the background graphic (if using one), the background-repeat is set to no-repeat, and padding is set to zero.

The inner <div> will specify margin properties to space it away from the edge of the outer div.

If I ever had to code for IE7 again, I would probably use a box-sizing polyfill instead. Another famous hack is the Tantek. For IE8, use DocType Switching.

The Easy Way Out

/* apply a natural box layout model to all elements */
 * { -moz-box-sizing: border-box; -webkit-box-sizing: border-box; box-sizing: border-box; }

Using the above CSS will totally change everything you just read. Instead of CSS properties width and height being the content area, it makes width and height include the border, padding, and content; which is the way we normally think it ought to work.

This will work for all browsers including IE8 and newer. This information came from Paul Irish.
