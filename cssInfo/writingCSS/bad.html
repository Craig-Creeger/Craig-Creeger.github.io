Additional Info

An example of a well constructed web page has been created so that you can compare the differences and learn from others’ mistakes.

This is a simple layout that contains the core features of most websites. We have a header, an area for navigation, a “main” content area, sidebar, and a footer. Take a look at the source code of this page and you will notice that:

  • There are unnecessary <div> tags
  • HTML5 structure tags are not used.
  • The navigation area is an unordered list. Once will often see this, but I do not think this is an appropriate use of a list.
  • Excess use of ID attributes and in-line styles.

If you compare the number of lines of HTML and CSS between the “good” page and “bad” page, you’ll see they are quite close, but as the size and complexity of the pages/site goes up, you will begin to save lines of code, and therefore sources of errors.
