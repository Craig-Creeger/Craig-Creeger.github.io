Where Do I Put the CSS?

CSS can be written as an external stylesheet, embedded, or inline.

External Stylesheets

External stylesheets are a separate file from your HTML. For example, your index.html” file would have the line of code,

<link href="global.css" type="text/css" rel="stylesheet">

inside the head tag. Then you would need a file called global.css” that contains all your CSS properties.

The styling you place in external stylesheets should be things that are common across your whole site. You will very likely always be using an external stylesheet for your pages.

Embedded Stylesheets

These are inside your HTML file. For example,

<html>
<head>
	<style type="text/css"> 
		h1 { color: DarkRed; }
	</style>
</head>
<body>
	<!-- your page goes here -->
</body
</html>

This works well during testing, when your site is only one page long, or for styling that is specific to only this page.

Inline Styles

These are element specific styles. For example,

<p style="width:80%; background-color:gainsboro; padding:.5em;"> This paragraph was style with an inline style.

Grid Style Sheets (GSS)

GSS reimagines CSS layout & replaces the browser's layout engine with one that harnesses the Cassowary Constraint Solver. This is a different way of writing style sheets and it solves problems that have plagued CSS for a long-time.

Best Practices

Use a CSS Reset

Because browsers rendering pages slightly differently, it is common to use a reset or a normalize file to make them consistent. Resets will remove all formatting (even bulleted lists no longer look like lists). A normalize will keep the best formatting, which makes it much more usable in my opinion.

Use http://necolas.github.io/normalize.css/ to start your sites.

Employ the best of OOCSS and Semantic CSS

Write the best HTML and CSS possible. Also, remove unused CSS either manually or using a tool.

Comment the sections and purpose of the classes/IDs

More likely than not, you will be maintaining the code months from now, and what made sense at the time, won’t after you have been away from it for awhile. Comments are your friend, so use them—a lot.

Organize Your Stylesheet

I like to break up the stylesheet into sections through the use of comment blocks (see example below).

html, body {
	margin:0; padding:0;
	font-family:Verdana, Geneva, sans-serif;
	line-height:1.4;
	font-size:100%;
}
/* =========================================================
   Typography
   ========================================================= */
.title { /* usually displayed in the <header> */
	 font-size:1.5em;
	 font-family: 'Iceberg', Georgia, "Times New Roman", Times, serif;
}
h1, h2, h3, h4, h5, h6 {
	 font-family: 'Iceberg', Georgia, "Times New Roman", Times, serif;
}
/* =========================================================
   Page Structure / Layout
   ========================================================= */
#wrapper {
	max-width:1024px;
	margin:0 auto;
}
header {
	background-color:#7a0019;
	color:#E3D3D3; /* light dusty red, used to thin the font */
}
/* =========================================================
   Misc Styling
   ========================================================= */
.developerNotes {
	background-color:#ffde7a;
	color:#300;
	font-family:Georgia, "Times New Roman", Times, serif;
	padding:0.6em;
	border:2px solid white;
	border-radius:10px;
	width:70%;
	margin:.5em auto;
	box-shadow:3px 3px 12px #5b0013, -8px -3px #b9b790;
}
.developerNotes:before {
	content:"Developer Notes:";
	display:block;
	font-weight:bold;
	margin-bottom:.5em;
	color:darkRed;
	border-bottom:thin solid darkRed;
}
.developerNotes:after {
	content:"--Craig  (555) 555-0457";
	display:block;
	padding-left:2em;
	color:darkRed;
}
.developerNotes a { color:hotpink; }
.developerNotes li {
	margin-left:1em;
}
.dataTable {
	border-collapse:collapse;
	margin:1em 0;
}
.dataTable thead {
	background-color:gainsboro;
	font-weight:bold;
}
.dataTable td,
.dataTable th {
	border:thin solid black;
	padding:0 .5em;
}
/* =========================================================
   Buttons
   ========================================================= */
.button { /* make link or <input submit> look like a button */
	text-decoration:none;
	color:black;
	display:inline-block;
	padding:0 1em;
	background-color:#ffcc33;
	border-radius:6px;
	border:thin solid #7a0019;
}

/* =========================================================
   Plugins or Vendor Styles
   ========================================================= 
    ColorBox Core Style:
    The following CSS is consistent between example themes and should not be altered.
*/
#colorbox, #cboxOverlay, #cboxWrapper {
	position:absolute; top:0;left:0; z-index:9999; overflow:hidden;
}
/* ============================================================
   Media Queries
   ============================================================ */
@media only screen and (max-width:1024px) {
	#wrapper {
		margin-top:0;
	}
}

@media only screen and (max-width:768px) {
	body {
		font-size:87.5%;
	}
	.title {
		font-size:1em;
	}
}
@media
	(-webkit-min-device-pixel-ratio: 2),
	(min-resolution: 192dpi) {
	    /* Retina-specific stuff here */
}
